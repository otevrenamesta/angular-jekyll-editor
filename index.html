Angular based editor for jekyll based sites

This is an angular based editor for jekyll based sites. Its intended for use with sites hosted on github pages. Github pages are awsome project but so less people can handle git and/or jekyll details. They just want to be able to perform few basic operations without any technical knowledge. This is goal of the angular-jekyll-editor.

usage

After login process (use your github credentials, that has access to target repo) you get to dashboard. It shows history of changes + some little info paragraps. Use menu to navigate to desired manipulation views.

Angular-jekyll-editor lets you handle 3 basic things:

  • pages: on page view you can see tree structure of pages along with handling buttons. On right side there is preview (uses iframe, so what you see is live version). See it! By clicking on a page tree item you change url of the iframe so you see the appropriate page. By click on pencil sign, you get to page edit form with content of the selected page. You can change whatever you want. See it! Submit the form for apply the changes or cancel the edits. Both action will lead you back to page view. By clicking on the plus sign, you add subpage of the appropriate page (through page edit form).
  • posts: posts are very similar to pages except they are not structured in tree. They instead follow category/tag based structure. That's why it is necessary to fill then when creating new posts. See it!
  • media files: And to upload/delete media files. Just filebrowsing button and list of uploaded files (with delete button). See it! What to say more about? :)

Mardown authoring and link editor

Markdown is hell simple markup language that everybody can handle. For link and image tags there is link editor, that simplify actual link creation. When you edit link, writing into link input starts suggesting the options based on its current content (typeahead). It offers you existing articles and media files. On right part, there is tree of existing pages. Double-click will set the link input with desired page address. See it!
